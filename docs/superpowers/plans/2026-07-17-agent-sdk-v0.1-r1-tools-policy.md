# Agent SDK v0.1 R1 Built-in Tools and Policy Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:test-driven-development for each task and superpowers:systematic-debugging for any unexpected failure. Use superpowers:verification-before-completion before recording the R1 checkpoint.

**Goal:** Ship built-in `read`, `write`, and `bash` Tools that use the ordinary Tool pipeline and enforce configurable workspace-scoped `allow`, `deny`, and `ask` decisions.

**Architecture:** Add immutable permission rules evaluated by the existing `PolicyEngine`, and register three ordinary `ToolSpec`/handler pairs during SDK initialization. Handlers load Session workspace roots, canonicalize every path before I/O, bound output, and rely on the existing permission broker for user decisions.

**Tech Stack:** Python 3.13, asyncio subprocesses, pathlib, Pydantic v2, existing ToolRegistry/ToolExecutor/PermissionBroker/StateStore, pytest, pytest-asyncio.

## Global Constraints

- Built-ins are normal registered Tools; do not add a separate execution path.
- Preserve application Tool registration and MCP behavior.
- A path outside all configured workspace roots is denied before filesystem/process work.
- An `allow` permission rule never overrides workspace containment.
- Explicit deny wins over allow/ask; otherwise the most-specific matching rule wins.
- `ask` continues through the existing permission request/resolution API.
- `bash` receives an argv array, not an interpolated shell string.
- All text and process output is bounded; Trace must not receive unbounded stdout/stderr.
- Production SDK construction enables built-ins by default. Tests may explicitly disable them to preserve narrow existing assertions.
- R1 does not promise a hardened OS sandbox; it promises path/cwd containment and command policy.

```python
class PermissionRule(BaseModel):
    model_config = ConfigDict(frozen=True, extra="forbid")

    outcome: PermissionOutcome
    tool: str = "*"
    path_prefix: Path | None = None
    command_prefix: tuple[str, ...] = ()

    @field_validator("tool")
    @classmethod
    def validate_tool(cls, value: str) -> str:
        if not value:
            raise ValueError("tool cannot be empty")
        return value


@dataclass(frozen=True)
class RuleMatch:
    rule: PermissionRule
    specificity: tuple[int, int, int]


def match_rule(rule: PermissionRule, request: PermissionRequest) -> RuleMatch | None:
    if rule.tool not in {"*", request.tool_name}:
        return None
    path_score = _match_path(rule.path_prefix, request.arguments.get("path"))
    if path_score is None:
        path_score = _match_path(rule.path_prefix, request.arguments.get("cwd"))
    if rule.path_prefix is not None and path_score is None:
        return None
    argv = request.arguments.get("argv")
    if rule.command_prefix and (
        not isinstance(argv, tuple | list)
        or tuple(argv[: len(rule.command_prefix)]) != rule.command_prefix
    ):
        return None
    return RuleMatch(
        rule=rule,
        specificity=(
            int(rule.tool != "*"),
            path_score or 0,
            len(rule.command_prefix),
        ),
    )
```

Implement `_match_path` using resolved, normalized path parts and
`Path.is_relative_to`; never use string prefix matching.

- [ ] **Step 3: Implement one shared resolver**

```python
def resolve_workspace_path(
    roots: Sequence[Path],
    requested: str | Path,
    *,
    for_write: bool,
) -> Path:
    if not roots:
        raise ToolAccessDenied("session has no workspace")
    candidate = Path(requested)
    candidates = (
        (candidate,) if candidate.is_absolute()
        else tuple(root / candidate for root in roots)
    )
    for root, raw in _pair_candidates(roots, candidates):
        canonical_root = root.resolve(strict=True)
        canonical = _resolve_with_existing_parent(raw) if for_write else raw.resolve(strict=True)
        if canonical.is_relative_to(canonical_root):
            return canonical
    raise ToolAccessDenied("path is outside configured workspace")
```

`_resolve_with_existing_parent` must walk upward to the nearest existing parent,
resolve that parent, then append only normal remaining path components. Reject
empty names, `.`/`..` remnants, and Windows alternate-stream components containing
`:` after the drive component.

Define private execution markers:

```python
class ToolAccessDenied(Exception):
    pass

    pass
```

`ToolExecutor` maps them to `ToolResultStatus.DENIED` and
`ToolResultStatus.TIMED_OUT` respectively, using the existing bounded normalized
error result and events. It never exposes the exception type or a filesystem path

- [ ] **Step 3: Implement file handlers**

```python
async def read_file(path: str, max_bytes: int | None = None, *, context: ToolContext) -> dict[str, object]:
    roots = await workspace_roots(store, context.session_id)
    target = resolve_workspace_path(roots, path, for_write=False)
    limit = min(max_bytes or output_limit, output_limit)
    preview, truncated = await asyncio.to_thread(_read_prefix, target, limit)
    return {
        "path": relative_display_path(target, roots),
        "content": preview.decode("utf-8", errors="replace"),
        "truncated": truncated,
        "bytes_read": len(preview),
    }


async def write_file(
    path: str,
    content: str,
    overwrite: bool = False,
    *,
    context: ToolContext,
) -> dict[str, object]:
    roots = await workspace_roots(store, context.session_id)
    target = resolve_workspace_path(roots, path, for_write=True)
    encoded = content.encode("utf-8")
    await asyncio.to_thread(_atomic_write, target, encoded, overwrite)
    return {"path": relative_display_path(target, roots), "bytes_written": len(encoded)}
```

Write to a unique temporary file in the same directory, flush/close it, and replace
the target only after all validation succeeds. Remove only that owned temporary
file on failure.

`_read_prefix` opens the file in binary mode and reads at most `limit + 1` bytes;
it must not use `Path.read_bytes()`.

- [ ] **Step 4: Implement argv-based bash**

```python
process = await asyncio.create_subprocess_exec(
    *argv,
    cwd=str(canonical_cwd),
    stdout=asyncio.subprocess.PIPE,
    stderr=asyncio.subprocess.PIPE,
)
assert process.stdout is not None and process.stderr is not None
stdout_task = asyncio.create_task(_drain_bounded(process.stdout, output_limit))
stderr_task = asyncio.create_task(_drain_bounded(process.stderr, output_limit))
try:
    await asyncio.wait_for(
        process.wait(),
        timeout=min(timeout_seconds or default_timeout, maximum_timeout),
    )
except TimeoutError:
    process.kill()
    await process.wait()
    await asyncio.gather(stdout_task, stderr_task)
    raise ToolExecutionTimedOut("bash command timed out")
stdout, stdout_truncated = await stdout_task
stderr, stderr_truncated = await stderr_task
```

Reject empty argv and NUL characters. Do not use `shell=True`. Bound combined
stdout/stderr to the configured byte limit and return decoded replacement text.
`_drain_bounded` continues draining after the limit while discarding excess bytes
so the subprocess cannot block on a full pipe and SDK memory remains bounded.
