# Agent SDK v0.1 R3 Automatic Context L0-L4 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:test-driven-development for each task, superpowers:systematic-debugging for model/recovery failures, and superpowers:verification-before-completion before the R3 checkpoint.

**Goal:** Build and apply a durable Context View before every model request, automatically select real L0-L4 strategies, preserve exact recovery requests, and compose default/application/Skill system prompts into a traceable Prompt Manifest.

**Architecture:** Convert Session events plus the current Run checkpoint into immutable source messages. Pure L0-L2 strategies render deterministic messages; LiteLLM-backed L3/L4 create validated capsules and fall back to L2. A `ContextMiddleware` is the single RunEngine hook that prepares provider messages while leaving checkpoint ledger messages unchanged. The exact prepared request is stored with the external operation for recovery.

**Tech Stack:** Python 3.13, asyncio, Pydantic v2, LiteLLM, existing ContextPlanner/ContextCompactor/PromptComposer/SkillRegistry/RunEngine/StateStore, pytest, pytest-asyncio.

## Global Constraints

- Run Context planning before every new provider model call, including calls after Tool results.
- Do not mutate or delete Session events.
- L1/L2 make no model call.
- L3/L4 use LiteLLM structured output with purpose `context_compaction`.
- L3/L4 failure, invalid schema, timeout, or over-budget output falls back to L2 and does not fail the main Run.
- Preserve current Run tool-call/tool-result message structure exactly.
- Do not feed prepared system/capsule messages back into the Run checkpoint; prepare afresh from durable sources.
- Persist exact prepared model requests for safe recovery; public Trace events contain ids/hashes, not raw prompt text.
- Automatic level selection applies the recommended level. A forced level is an explicit override, not the default behavior.
- System prompt defaults to the packaged profile and may be extended by an application prompt.
- Activated Skills are immutable prompt layers and appear in the Prompt Manifest.
- No embeddings, vector search, filesystem Artifact dependency, or arbitrary retrieval plugin is added in R3.

- [ ] **Step 4: Implement L0/L1/L2**

```python
@dataclass(frozen=True)
class StrategyResult:
    items: tuple[SourceMessage, ...]
    source_refs: tuple[str, ...]
    transformations: tuple[str, ...]


def render_level(
    level: CompactionLevel,
    sources: tuple[SourceMessage, ...],
    *,
    recent_messages: int,
    tool_preview_bytes: int,
) -> StrategyResult:
    if level is CompactionLevel.L0:
        return apply_l0(sources)
    if level is CompactionLevel.L1:
        return apply_l1(sources, tool_preview_bytes=tool_preview_bytes)
    if level is CompactionLevel.L2:
        return apply_l2(
            sources,
            recent_messages=recent_messages,
            tool_preview_bytes=tool_preview_bytes,
        )
    raise ValueError("deterministic renderer supports L0-L2 only")
```

Use canonical JSON hashes for duplicate Tool detection. Use UTF-8 byte-safe
head/tail truncation. L2 outcomes are JSON strings with stable keys:
`kind`, `role`, `status`, `summary`, `source_refs`.

def serialize_model_request(request: ModelRequest) -> dict[str, Any]:

def deserialize_model_request(value: Mapping[str, Any]) -> ModelRequest:
    validated = _ModelRequestPayload.model_validate(value)
    return ModelRequest(
        model=validated.model,
        messages=tuple(dict(message) for message in validated.messages),
        tools=tuple(dict(tool) for tool in validated.tools),
        params=dict(validated.params),
        purpose=validated.purpose,
