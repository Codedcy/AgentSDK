# Agent SDK v0.1 R5 Trace Attribution and Release Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:test-driven-development for behavior tasks, superpowers:requesting-code-review for the whole-v0.1 review, and superpowers:verification-before-completion before claiming or tagging `0.1.0`.

**Goal:** Expose normalized live/historical Trace data, deterministic evidence-linked Run attribution, the agreed basic cross-Run metrics, and a verified installable Agent SDK `0.1.0`.

**Architecture:** Keep the immutable event log as source of truth. A read-only Trace projector normalizes stage facts from events/snapshots; an attribution projector joins Context source refs, Tool/Child outcomes, evaluation records, and terminal status. No LLM is used for attribution. Finish the growing acceptance scenario, reference example, release docs, package version, and one release-candidate gate.

**Tech Stack:** Python 3.12-3.13, asyncio, Pydantic v2, existing Event/Query/Subscription/Evaluation/Analytics services, pytest, Ruff, strict mypy, hatchling/build, uv.

## Global Constraints

- Trace collection is always SDK-side; the application decides whether/when/how to display it.
- Public Trace payloads use ids, hashes, bounded previews, usage, sanitized errors, and evidence refs; do not expose credentials or unbounded raw values.
- Attribution is deterministic contribution/correlation, never causal proof.
- A completed root Run has no terminal failure attribution; failed Child/Tool stages remain contributors and may produce hints.
- A failed/interrupted root Run uses the first terminal failing stage in cursor order across its execution tree.
- A Tool result is consumed only when a later Context View references its completion event/result id; otherwise it is unused.
- A Child result/message is consumed only when a parent Context View references its result/message id.
- Existing `success_rate`, `tool_failures`, and `tool_failure_rate` remain the v0.1 cross-Run metrics.
- Aggregate Tool usefulness ranking, multidimensional failure taxonomy, confidence/statistics, generated insights, and causal analysis remain post-v0.1.
- Do not publish to an external package registry without a separate user instruction.

## Public Interfaces

```python
class TraceStageKind(StrEnum):
    RUN = "run"
    STEP = "step"
    CONTEXT = "context"
    MODEL = "model"
    TOOL = "tool"
    PERMISSION = "permission"
    WORKFLOW = "workflow"
    WORKFLOW_NODE = "workflow_node"
    CHILD = "child"
    MESSAGE = "message"
    EVALUATION = "evaluation"
    RECOVERY = "recovery"


class TraceStageStatus(StrEnum):
    RUNNING = "running"
    WAITING = "waiting"
    COMPLETED = "completed"
    FAILED = "failed"
    DENIED = "denied"
    TIMED_OUT = "timed_out"
    INTERRUPTED = "interrupted"


class TraceStage(BaseModel):
    stage_id: str
    kind: TraceStageKind
    entity_id: str
    parent_stage_id: str | None
    run_id: str
    session_id: str
    status: TraceStageStatus
    started_at: datetime | None
    ended_at: datetime | None
    duration_ms: float | None
    input_refs: tuple[str, ...] = ()
    output_refs: tuple[str, ...] = ()
    usage: TokenUsage | None = None
    cost_usd: float | None = None
    error_code: str | None = None
    retryable: bool | None = None
    evidence_event_ids: tuple[str, ...]


class TraceTimeline(BaseModel):
    root_id: str
    root_kind: Literal["run", "workflow"]
    stages: tuple[TraceStage, ...]
    as_of_cursor: int
```

Attribution:

```python
class AttributionContributor(BaseModel):
    kind: Literal["model", "tool", "context", "workflow", "child", "evaluation"]
    entity_id: str
    status: str

class FailureAttribution(BaseModel):
    stage_id: str
    stage_kind: TraceStageKind
    code: str
    retryable: bool
    evidence_ids: tuple[str, ...]


class ImprovementHint(BaseModel):
    code: Literal[
        "repeated_tool_failure",
        "unused_tool_output",
        "context_fallback",
        "workflow_loop_limit",
        "child_failure",
        "permission_denied",
        "interrupted_external_work",

class AttributionSummary(BaseModel):

    terminal_status: RunStatus
    failure: FailureAttribution | None
    contributors: tuple[AttributionContributor, ...]
    evaluation_ids: tuple[str, ...]
    hints: tuple[ImprovementHint, ...]

```

Facade:

- [ ] **Step 6: Implement TraceService/facade**

- [ ] **Step 3: Implement contribution joins**

Load, as of the Trace high-water:

- `ContextView.source_refs`, `message_refs`, and `consumed_message_ids`;
- Prompt Manifest Context/View ids;
- Tool completion event/result ids;
- Child terminal run ids and Agent message ids;
- Workflow instruction/node ids;
- evaluation completion records.

Rules:

```python
consumed_refs = frozenset(
    ref
    for view in context_views
    for ref in (*view.source_refs, *view.message_refs, *view.consumed_message_ids)
)
tool_disposition = "consumed" if tool.result_ref in consumed_refs else "unused"
child_disposition = (
    "consumed"
    if child.result_ref in consumed_refs
    or any(message.message_id in consumed_refs for message in child.messages)
    else "unused"
)
```

Model outputs that precede a later model request are `consumed`; the final
successful model output is `terminal`. Context/Workflow/evaluation records are
`supporting` unless terminal.

- [ ] **Step 4: Implement failure and fixed hints**

If root status is completed, failure is `None`. Otherwise choose the first
failed/denied/timed-out/interrupted stage by evidence cursor; if none exists, use
the root terminal stage.

Generate one deduplicated hint per code:

- two or more failures of the same Tool: `repeated_tool_failure`;
- any unused successful Tool result: `unused_tool_output`;
- Context fallback: `context_fallback`;
- loop failure code: `workflow_loop_limit`;
- failed Child: `child_failure`;
- denied permission: `permission_denied`;
- interrupted external operation: `interrupted_external_work`.

Hint summaries are fixed package strings. Evidence ids are sorted by cursor.
