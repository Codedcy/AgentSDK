# Agent SDK v0.1 R2 Workflow Control Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:test-driven-development for every task, superpowers:systematic-debugging for recovery failures, and superpowers:verification-before-completion before recording the R2 checkpoint.

**Goal:** Extend Workflow with restricted conditions and bounded loops while preserving sequential Workflow compatibility and never repeating a completed Agent node after restart.

**Architecture:** Keep `WorkflowIR` as the durable class, add schema version 2 with a flattened Agent-node table plus immutable control instructions, and retain schema version 1 validation for existing snapshots. Compile nested definitions into a small program. A pure program reducer chooses the next action; `WorkflowState` atomically persists program counter, branch decisions, loop counters, and parsed node outputs.

**Tech Stack:** Python 3.13, Pydantic v2 discriminated unions/forward references, PyYAML safe loading, existing WorkflowState/WorkflowExecutor/Run recovery, pytest, pytest-asyncio.

## Global Constraints

- Supported node kinds are exactly `agent`, `condition`, and `loop`.
- Conditions read immutable Workflow inputs and completed node outputs only.
- Operators are exactly `eq`, `ne`, `gt`, `gte`, `lt`, `lte`, `contains`, and `exists`.
- Dotted paths and non-negative array indexes are allowed; Python attribute access/calls/eval are forbidden.
- Every loop has `1 <= max_iterations <= compiler_max_loop_iterations`.
- Nesting depth is statically bounded by `compiler_max_control_depth`.
- A loop-limit failure is visible and durable.
- Completed Agent node ids execute at most once, including after process restart.
- Existing schema-v1 sequential snapshots still load/resume.
- Model-generated YAML is compiled/validated but is not automatically started.
- Do not add parallelism, foreach, arbitrary retry, compensation, approval nodes, or subworkflows.

```python
WorkflowStep = Annotated[

class WorkflowDefinition(BaseModel):

    name: str

    edges: tuple[WorkflowEdge, ...] = ()   # legacy definition input
```

Exactly one of `steps` or `nodes` is non-empty. Legacy `nodes`/`edges` compile to a
v2 sequential instruction program; persisted `WorkflowIR(schema_version=1)` keeps
its current hash and validation behavior.

```python
class WorkflowInstruction(BaseModel):
    id: str
    op: Literal["agent", "branch", "loop_check", "jump", "complete"]
    agent_node_id: str | None = None
    expression: WorkflowExpression | None = None
    true_pc: int | None = None
    false_pc: int | None = None
    target_pc: int | None = None
    loop_id: str | None = None
    max_iterations: int | None = None


class WorkflowControlState(BaseModel):
    model_config = ConfigDict(frozen=True, extra="forbid")

    program_counter: int = 0
    revision: int = 1
    selected_branches: Mapping[str, Literal["then", "else"]] = Field(default_factory=dict)
    loop_iterations: Mapping[str, int] = Field(default_factory=dict)
    outputs: Mapping[str, JsonValue] = Field(default_factory=dict)
```

`WorkflowIR` retains its current fields and adds defaults:

```python
schema_version: Literal[1, 2] = 2
inputs: Mapping[str, JsonValue] = Field(default_factory=dict)
instructions: tuple[WorkflowInstruction, ...] = ()
```

For schema 1, inputs/instructions must be empty and the current hash algorithm is
unchanged. For schema 2, `edges` must be empty, `nodes` contains each compiled
Agent node once, `instructions` ends in `complete`, and the hash covers inputs,
nodes, and instructions.

```python
_SEGMENT = re.compile(r"^[A-Za-z0-9_-]+$")


def resolve_path(scope: Mapping[str, JsonValue], path: str) -> JsonValue:
    parts = path.split(".")
    if not parts or parts[0] not in {"inputs", "outputs"}:
        raise WorkflowExpressionError("path must start with inputs or outputs")
    current: JsonValue = scope
    for part in parts:
        if not _SEGMENT.fullmatch(part) or part.startswith("__"):
            raise WorkflowExpressionError("invalid path segment")
        if isinstance(current, Mapping):
            if part not in current:
                raise MissingWorkflowValue(path)
            current = current[part]
        elif isinstance(current, tuple):
            if not part.isdecimal() or int(part) >= len(current):
                raise MissingWorkflowValue(path)
            current = current[int(part)]
        else:
            raise MissingWorkflowValue(path)
    return current
```

Implement comparisons explicitly. Reject boolean-as-number comparisons and
non-finite values at model validation. `contains` accepts:

- string contains string;
- tuple contains JSON-equal value;
- mapping contains a string key.

Use one internal mutable builder only during compilation:

```python
@dataclass
class _InstructionDraft:
    id: str
    op: InstructionOp
    agent_node_id: str | None = None
    expression: WorkflowExpression | None = None
    true_label: str | None = None
    false_label: str | None = None
    target_label: str | None = None
    loop_id: str | None = None
    max_iterations: int | None = None


class _ProgramBuilder:
    def mark(self, label: str) -> None: ...
    def emit(self, instruction: _InstructionDraft) -> None: ...
    def compile_steps(self, steps: tuple[WorkflowStep, ...], depth: int) -> None: ...
    def freeze(self) -> tuple[WorkflowInstruction, ...]: ...
```

Compilation rules:

- Agent: emit `agent`; the executor advances to the next pc only after durable
  completion.
- Condition: emit `branch`, compile then, emit jump-to-join, compile else, mark join.
- Loop: mark check, emit `loop_check`, compile body, emit jump-to-check, mark exit.
- Workflow: compile steps, emit `complete`.

Validate unique ids before emitting. Resolve labels only in `freeze`; reject
missing/out-of-range targets. Freeze inputs through the shared JSON freezer.

```python
class ExecuteAgent:
```
